# SOKDAK 디자인 원칙

> SOKDAK은 기능만 되는 것이 아니라, **아름답고 즐거운** 제품이어야 한다.

## 핵심 원칙

### 1. 익명성이 안전하게 느껴져야 한다

- 시각적으로 '보호받고 있다'는 느낌 전달
- 부드러운 모서리, 따뜻한 색상, 차분한 그라데이션
- 아바타는 랜덤 생성 일러스트 (절대 실사 아님)
- 닉네임 표시 시 고정된 색상 뱃지 (해시 기반)

### 2. 재미가 자연스럽게 녹아들어야 한다

- 점심 투표, 밸런스 게임, 감정 공유 = 놀이처럼 느껴지는 UI
- 마이크로 인터랙션: 투표 시 파티클, 리액션 시 바운스
- 결과 시각화: 차트, 히트맵, 프로그레스 바를 아름답게
- 게이미피케이션 요소: 배지, 연속 출석, 랭킹

### 3. 심플 > 복잡

- 한 화면에 한 가지 목적
- 네비게이션은 최대 2-depth
- 모달보다 페이지 전환 선호
- 설정은 최소화 (합리적 기본값)

### 4. 모바일 우선

- 모든 UI는 모바일 기준으로 먼저 설계
- 터치 타겟: 최소 44px
- 스와이프 제스처 활용 (카드 넘기기, 삭제)
- 바텀 시트 > 드롭다운

### 5. 일관성

- shadcn/ui 기반 통일된 컴포넌트
- 색상/타이포/간격 토큰 시스템 준수
- 같은 행동 = 같은 패턴 (리스트, 카드, 폼)

## 감성 키워드

```
따뜻한 · 안전한 · 장난스러운 · 깔끔한 · 현대적인
```

## 안티패턴

- ❌ 기업용 대시보드처럼 보이는 UI
- ❌ 재미 없는 기능 나열
- ❌ 텍스트만 가득한 페이지
- ❌ 복잡한 설정 화면
- ❌ 90년대 게시판 느낌
