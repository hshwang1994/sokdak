# SOKDAK UX 우선순위

## P0: 첫 경험 (First-time Experience)

사용자가 SOKDAK을 처음 접했을 때 3분 안에 가치를 느껴야 한다.

### 가입 플로우

```
이메일 입력 → OTP 확인 → 닉네임 설정 → 완료 (3단계)
```

- 가입 30초 이내 완료 가능
- 가입 직후 오늘의 감정 선택 화면 표시 (즉시 참여감)
- 인기 게시글 3개 자동 노출

### 핵심 플로우 (가입 후 1분 내)

1. 오늘의 감정 선택 (1탭)
2. 인기 게시글 읽기 (스크롤)
3. 리액션 남기기 (1탭)

## P1: 일상 사용 패턴

### 점심 투표 (11:00-11:30)

```
[알림] "오늘 점심 뭐 먹을까요?" → 후보 3-5개 → 1탭 투표 → 실시간 결과
```

- 푸시 알림 → 앱 진입 → 투표 → 결과 확인: 10초 이내
- 결과는 재미있는 시각화 (파이 차트 + 1위 애니메이션)

### 게시글 작성

```
+ 버튼 → 카테고리 선택 → 제목/본문 → 게시 (3단계)
```

- 마크다운 에디터 (미리보기 토글)
- 이미지 드래그앤드롭
- 임시 저장 자동

### 댓글 참여

- 게시글 하단 댓글 영역
- 대댓글 1-depth (indent)
- 익명 아바타 자동 생성

## P2: 재미 기능

### 밸런스 게임

```
카드 스와이프 (A vs B) → 통계 → 댓글 토론
```

- 텐더 스타일 스와이프 UX
- 선택 후 % 통계 즉시 공개
- 의견 댓글 가능

### 고민 상자

```
고민 작성 → 익명 답변 대기 → 채택
```

- 따뜻한 UI (손글씨 느낌 폰트 옵션)
- 답변 알림
- 채택 시 감사 배지

### 감정 히트맵

```
전체 조직의 오늘 감정 분포 → 히트맵 시각화
```

- 컬러풀한 그리드
- 시간대별 변화 추이
- 개인 감정 기록 아카이브

## P3: 관리 기능

- Admin 대시보드: 심플, 데이터 중심
- 신고 처리: 인라인 액션 (숨김/삭제/경고)
- 사용자 관리: 검색 + 필터 + 제재 이력

## UX 메트릭

| 메트릭 | 목표 |
|--------|------|
| 가입 완료 시간 | < 30초 |
| 첫 리액션까지 | < 60초 |
| 점심 투표 참여율 | > 70% |
| 일일 재방문율 | > 40% |
| 페이지 로드 시간 | < 1초 |
| 에러율 | < 0.1% |
